網站前端程式碼標準 & 實務典範

概覽

本文件描述 Isobar US 針對前端開發實務上所建立的 Web 應用程式規範指南。這份指南是為了讓任何想快速了解我們實務經驗中各項進化的人所寫。

撰寫這份文件的主要動機有二：1) 保持程式碼一致性，以及 2) 列出最佳實務。藉由維持程式碼在撰寫樣式及公約上的一致性，我們可以減輕維護舊程式的負擔，且減少當前程式對於未來開發不利的情形；而依循最佳實務，我們可以確保網頁載入、效能等最佳化，程式碼也易於維護。

一般準則

前端開發的支柱

  1. 分離展示、內容、互動行為。
  2. 標籤應符合格式、語意正確地標示，大致上也應通過驗證。
  3. Javascript 應用以逐步增進使用經驗。

一般實務

縮排

無論寫什麼程式語言，我們都要求以 soft tab（以空白字元代替 tab 字元）的方式來縮排。在你的編輯器上，按下 Tab 鍵應等同於 4 個空白字元。

易讀與壓縮

當維護現存檔案時，相較於節省檔案大小、我們更重示程式碼易讀性。我們鼓勵使用空白字元、甚至在適當時機用點 ASCII Art。開發者不必刻意壓縮 HTML 或 CSS，也不需為 JavaScript 編碼。

我們會以伺服端技術，自動壓縮、gzip 所有 CSS 及 JavaScript 等靜態檔案。
